내부 클래스(Inner Class) -> 잘 사용하진 않음
	하나의 클래스에서 a작업과 b작업이 있을 때에는 따로 분리하여 클래스를 만들지 않고, 클래스 안에 클래스를 선언하여 설계한다. 
	이 때 밖에 있는 클래스는 외부 클래스 안에 있는 클래스는 내부 클래스
	외부 클래스가 메모리에 할당되어야 내부 클래스를 객체화할 수 있기 때문에 클래스를 숨기기 위해 내부 클래스를 사용하기도 하며
	이를 "은닉화"라고 한다. 내부 클래스는 외부 클래스의 필드이기 때문에 외부 클래스의 필드를 자신의 필드처럼 가져다 사용할 수 있다.
	* 메소드 안에서 클래스를 선언할 수 있다.

익명 클래스(Anonymous Inner Class) -> 
	이름이 없는 클래스이며 구현되지 않은 필드를 구현하기 위해 일회성으로 생성되는 클래스이다.

함수형 인터페이스
	인터페이스 중 추상 메소드를 하나만 가지고 있는 인터페이스를 함수형 인터페이스라고 한다.
	이 때 @FunctionalInterface를 인터페이스 위에 작성하여 단 하나의 "추상 메소드만" 선언할 수 있도록 제한.
	
람다식(Lambda Expression) -> 웹 개발 시 많이 사용됨!
	이름이 없는 메소드로서 변수처럼 사용이 가능하며, 매개변수로도 전달이 가능하다.
	함수형 인터페이스는 추상 메소드가 한 개만 선언되기 때문에 메소드 이름이 필요없다.
	따라서 람다식을 익명 메소드(Anoymous Method)라고도 부른다.
